Admin Setup

Keep the backend lightweight, practical, and easy to maintain.

This page is for the configuration layer only: GitHub Pages, Apps Script, Google Sheets, and the browser endpoint used for draft and final submission.

Assessment users should not need this page.

It exists to keep backend setup out of the main response flow while still staying easy to access for administrators.

System Stack

Three simple layers handle hosting, submission, and storage.

GitHub Pages

Hosts the static site, front-end logic, and live assessment interface.

Apps Script

Receives draft and final payloads from the browser and writes them to Sheets.

Google Sheets

Stores drafts, completed sessions, question responses, and priority output.

Browser Endpoint

Configure the Apps Script URL used by this browser.

Use this if you want to test draft autosave and final submission directly from the live site.

No Apps Script endpoint configured yet.

Recommended tabs
  • Assessment_Drafts: one row per autosaved draft session.
  • Assessment_Sessions: one row per submitted session.
  • Assessment_Responses: one row per question response.
  • Assessment_Priority: ranked output by session.

Setup Flow

Use a short admin sequence and then leave the page.

01 Deploy Apps Script

Bind the script to your target spreadsheet and publish it as a web app.

02 Save the endpoint

Paste the `/exec` URL into this page so the current browser knows where to write.

03 Run a draft test

Open the assessment, answer a few questions, and confirm the draft row appears.

04 Submit a full session

Complete one run and verify sessions, responses, and priority rows appear correctly.
