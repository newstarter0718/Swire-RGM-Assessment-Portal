Why This Exists

One enterprise assessment that explains both maturity and management action.

The methodology deck, workbook, and assessment manual all point to one executive question: can each market systematically convert RGM opportunity into sustained value creation?

  • Connect market assessment to strategic ambition and profitable growth.
  • Separate business performance diagnosis from capability maturity diagnosis.
  • Expose whether the bottleneck sits in insight, strategy, policy, execution, or sustainability.
  • Create a repeatable cycle for reassessment, calibration, and action planning.

Assessment Logic

Answers become priorities through a very explicit scoring chain.

This should feel more like a management system than a survey. Each question feeds a score, each score is read against a target, and each gap contributes to the final priority list.

01

Respond

Complete the maturity questions by pillar and anchor each score with evidence notes or comments.

02

Score

Each answer is translated into a maturity score that rolls into pillar, stage, and enabler averages.

03

Compare

Actual maturity is read against target state so the system can show where the largest gaps sit.

04

Prioritize

Weighted gap-to-target logic produces a ranked list of issues that matter most for follow-through.

Decision Principle

Not every low score becomes a top priority.

Priority is driven by weighted maturity gap to target, so the portal highlights where shortfalls are both meaningful and strategically important instead of simply listing every weak answer.

Operating Cycle

Designed as a recurring management rhythm, not a one-off survey.

The assessment is only useful if it can be repeated, calibrated, and translated into action. The cycle below shows how the tool fits into that broader operating model.

1 Fact base and pre-read

Build the market context and prepare evidence before self-scoring begins.

2 Market self-assessment

Score the detailed pillar questions with evidence-backed maturity anchors.

3 Interview and validation

Test whether the self-scores match actual decisions, routines, and governance quality.

4 Calibration

Moderate scores, align evidence standards, and confirm the final maturity picture.

5 Report and roadmap

Translate the dashboard into priorities, ownership, and an action roadmap.

6 Pulse and follow-up

Use lighter annual reviews and quarterly tracking to keep progress visible between full cycles.

Delivery Architecture

Simple web delivery in front, lightweight assessment logging behind.

The admin layer stays intentionally lightweight. GitHub Pages hosts the experience, Apps Script handles submission, and Google Sheets becomes the working admin console.

1. GitHub Pages

Hosts the narrative site, assessment experience, scoring logic, and results dashboard.

2. Apps Script web app

Receives the browser submission payload and writes the structured output into Sheets.

3. Google Sheets

Acts as a lightweight admin environment for sessions, question responses, and priority outputs.

Recommended sheet tabs

  • Assessment_Sessions: one row per completed assessment with summary metrics.
  • Assessment_Responses: one row per question response with score, evidence, and comment.
  • Assessment_Priority: top priorities for each submitted session.

Current release scope

  • Core maturity questionnaire from the four pillar sheets
  • Live scoring and result dashboard
  • Google Sheets logging pattern
  • Knowledge, interview, and case modules reserved for later expansion
